# Workflow Overview

## Core Workflows

### 1. Product Browsing Flow

```mermaid
sequenceDiagram
    participant User
    participant ShopController
    participant Product Model
    participant Blade View

    User->>ShopController: GET /shop
    ShopController->>Product Model: fetchProducts()
    Product Model-->>ShopController: products collection
    ShopController-->>Blade View: shop.blade.php
    Blade View-->>User: Product listing page

    User->>ShopController: GET /product/{slug}
    ShopController->>Product Model: findBySlug(slug)
    Product Model-->>ShopController: product with variants
    ShopController-->>Blade View: details.blade.php
    Blade View-->>User: Product detail page
```

### 2. Add to Cart Flow

```mermaid
sequenceDiagram
    participant User
    participant CartController
    participant Cart
    participant Session

    User->>CartController: POST /cart/add
    CartController->>Cart: add(product, qty)
    Cart->>Session: persist cart data
    Cart-->>CartController: success
    CartController-->>User: redirect to cart
```

### 3. Checkout & Order Flow

```mermaid
sequenceDiagram
    participant User
    participant CartController
    participant Order Model
    participant PaymentController
    participant VNPay

    User->>CartController: GET /checkout
    CartController-->>User: checkout form

    User->>CartController: POST /place-an-order
    CartController->>Order Model: create order
    Order Model-->>CartController: order created
    CartController->>PaymentController: initiate payment
    PaymentController-->>VNPay: request payment URL
    VNPay-->>PaymentController: payment URL
    PaymentController-->>User: redirect to VNPay

    User->>PaymentController: VNPay return
    PaymentController->>Order Model: update status
    Order Model-->>PaymentController: confirmed
    PaymentController-->>User: order confirmation page
```

### 4. Wishlist Flow

```mermaid
sequenceDiagram
    participant User
    participant WishlistController
    participant Session

    User->>WishlistController: POST /wishlist/add
    WishlistController->>Session: add product to wishlist
    WishlistController-->>User: success response

    User->>WishlistController: GET /wishlist
    WishlistController-->>User: wishlist view

    User->>WishlistController: POST /wishlist/move-to-cart/{id}
    WishlistController->>Cart: move item
    WishlistController->>Session: remove from wishlist
    WishlistController-->>User: redirect to cart
```

### 5. Admin Product Management Flow

```mermaid
sequenceDiagram
    participant Admin
    participant AdminController
    participant Product Model
    participant Image Intervention

    Admin->>AdminController: GET /admin/product/add
    AdminController-->>Admin: product-add form

    Admin->>AdminController: POST /admin/product/store
    AdminController->>Image Intervention: process images
    Image Intervention-->>AdminController: image paths
    AdminController->>Product Model: create product
    Product Model-->>AdminController: product saved
    AdminController-->>Admin: redirect to products list
```

## Error Handling

- **Form Validation**: Laravel validation with error messages displayed in views
- **404 Handling**: Laravel automatic 404 for missing models
- **Payment Failures**: Redirect back to checkout with error message
- **Image Upload Failures**: Catch exceptions, display status message

## State Management

- **Cart State**: Stored in session via ShoppingCart package
- **Wishlist State**: Stored in session
- **User Authentication**: Laravel's built-in auth with session
- **Flash Messages**: Session-based status messages for redirects
